# Extract

## Summary
This content explores personal knowledge management and explicit memory systems, highlighting projects like Farzapedia (a personal Wikipedia from 2500 diary entries) and VaultMind (browser-based Karpathy wiki). Key themes include data ownership, local storage with markdown interoperability, recursive summarization for documents, and visualization of memories as navigable knowledge graphs.

## Key Points
- Farzapedia: built personal Wikipedia from 2500 diary entries, Apple Notes, and iMessage conversations, generating 400 detailed articles
- Karpathy emphasizes explicit, navigable memory with data ownership on local computer in simple markdown files
- Recursive summary tree approach for PDFs converts pages to MD, summarizes each, then summarizes summaries - preserving document structure
- VaultMind implements Karpathy wiki pattern in browser with zero backend (single HTML), supporting PDF drops, URLs, voice memos, and file drag with live knowledge graph
- LOTR Memory Explorer converted 480,000 words into 20,000 navigable memories with visual graph - demonstrating explicit memory at scale
- Obsidian integration works with YAML frontmatter and wiki links, providing graph view and Unix-style file interoperability
