# Extract

## Summary
Andrej Karpathy describes using LLMs to build personal wiki-style knowledge bases instead of traditional RAG systems. The approach emphasizes local data storage, simple markdown files, and incremental knowledge updates across sources and concepts directories, as demonstrated by Farzapedia's transformation of 2500 personal entries into 400 articles.

## Key Points
- Replace RAG with continuous wiki-style knowledge base maintained by LLMs
- Four key principles: explicit (visible/navigable), yours (local data), file over app (markdown), BYOAI
- Directory structure separates raw sources from structured concepts
- LLM incrementally updates concepts without overwriting existing knowledge
- Farzapedia example: 2500 diary entries, Apple Notes, and iMessages converted to 400 detailed articles
